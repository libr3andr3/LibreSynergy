# Changelog

All notable changes to LibreSynergy are documented here. The format follows
[Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and the project
adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.2.0] — 2026-07-07

A scope-and-trust release: the distribution now contains exactly the pieces a
creator or community needs to run their own livestream, courses, chat, and
payments — every service's source in the repo, every default neutral.

### Added

- **membership-sync and events sources** (`apps/membership_sync/`,
  `apps/events/`) now ship with the repo. Previously the payments unit said
  "copy them from your running instance" — unacceptable for code that moves
  people in and out of a paid tier. Both are short, pure-stdlib, and readable.
- **`ls init` renders the branded pages** — templates under `www/` render at
  init (previously only via `ls brand`), so a fresh install serves your brand,
  not placeholder files.
- Payments/membership/events services now receive their hostnames, brand
  name, price, and group names from `libresynergy.env` via compose — no
  per-app configuration to keep in sync.

### Changed

- **Creator-first README** — the front door now speaks to the person whose
  livestream, course, and paid community this runs, without dropping the
  community/municipality story.
- Secrets examples (`secrets/*.example`) now use exactly the variable names
  the services read (`AUTHENTIK_URL`, `ANNOUNCE_ROOM`, `EVENTS_ADMIN_TOKEN`,
  Frappe API keys), so following the examples produces a working install.
- All personal/flagship defaults removed from code and docs: hardcoded
  domains, example IPs, container names, watermark text, and the internal
  codename for the relay pattern. Rendered brand artifacts (`www/index.html`
  and friends) are no longer tracked — only their `*.template.*` sources are.
- `deploy-node.sh` requires an explicit `NODE_DIR` instead of assuming the
  flagship's paths; `ls-route` derives the Caddy container from the compose
  project name.

### Removed

- **Minecraft PoC** (`profile: games`), its docs, and its DNS automation —
  out of scope for the distribution. The raw-TCP routing pattern it
  demonstrated is documented with the livestream RTMP ingest instead.
- **Operator MCP tool** (`tools/verypowerful-mcp.py`) — flagship-operations
  tooling, not part of the community stack.
- **Personal jobs-board service** from the edge unit and Caddyfile — its
  source was never in the repo, so a fresh clone could not build it.

## [1.1.0] — 2026-07-05

Everything the flagship instance (yaya.sh) proved in production, folded back
into the portable distribution — plus the peer-to-peer sovereignty layer.

### Added

- **Passwordless auth** — email security-code signup and login flows for
  Authentik (usernames sync to Matrix; no passwords to leak).
- **Livestreaming** (`profile: stream`) — OwnCast unit: RTMP ingest, HLS out,
  live chat; embeddable in the app shell. `docs/livestreaming.md`.
- **Payments, both rails** — Stripe (card) alongside BTCPay (BTC + Monero),
  both settling to the same Authentik entitlement group.
- **Admin dashboard** (`profile: admin`) — first-party panel behind SSO for
  branding, sponsors, site banner, and Matrix announcements.
- **Course VOD swarm** (`profile: vod`) — sovereign WebTorrent tracker; static
  lessons peer-seeded PeerTube-style with an always-available origin webseed.
  `docs/peertube-layer.md`.
- **The mesh** (`mesh/`) — Ed25519 content signing for creator provenance, a
  STUN-style NAT-traversing rendezvous, and a segment relay. Web-onboarding →
  P2P funnel; measured crypto overhead in `docs/architecture-mesh.md`.
- **Operator MCP tool** (`tools/verypowerful-mcp.py`) — Cloudflare DNS, relay
  SNI/L4 routes, and Caddy vhosts as one AI-operator tool surface.
- **DNS automation** (`scripts/cf-dns-sync.sh`) — idempotently create every
  host record in a Cloudflare zone (DNS-only; TLS stays on your node).

## [1.0.0] — 2026-07-03

First portable release. LibreSynergy graduates from a proven single-instance
deployment to a white-label distribution: any community can now install it
on its own domain and hardware with one config file and one command.

### Added

- **White-label portability.** Every host path, domain, email, and address
  is parameterized (`LS_DATA_DIR`, `LS_WWW_DIR`, `LS_SECRETS_DIR`,
  `LS_BASE_DOMAIN` and per-service hostnames, `LS_ADMIN_EMAIL`, `LS_WG_IP`,
  `LS_RELAY_WG_IP`, `LS_TZ`). Nothing instance-specific remains in the
  distribution.
- **Secrets scaffolding.** All credentials moved out of compose files into
  `${LS_SECRETS_DIR}/*.env` (mode 600), generated by `ls init`; the repo
  ships only `secrets/*.env.example` templates with generation instructions.
- **Generic TCP/UDP routing.** `ls-route` manages both hostname-routed web
  services (Caddy vhost + relay SNI map) and port-routed stream services
  (relay nginx stream to a WireGuard-bound port on the node), keeping both
  machines' configs generated from one source.
- **Minecraft proof of concept** for stream routing: binds `${LS_WG_IP}:25565`,
  relayed as raw TCP through the VPS, enabled via the `games` profile.
- **Compose profiles** for optional features: `payments` (BTCPay bridge,
  membership sync, events), `files` (Filebrowser), `games` (Minecraft).
  Core services (edge, SSO, chat, classroom, webinars) are always on.
- **Bootstrap and lifecycle tooling.** `scripts/bootstrap-vps.sh`,
  `scripts/bootstrap-node.sh`, and the `ls` CLI (`init`, `up`, `down`,
  `doctor`) take two fresh machines to a live community in about 30 minutes.
- **Full documentation set:** quickstart, architecture, add-a-service
  runbook (web and TCP), operations (backups, updates, rotation, known
  gotchas), and an honest security model.

### Changed

- The canonical loopback port map (Authentik 8300/9443, Synapse 8008,
  Frappe LMS 8100, Jitsi 8200, Cinny 8114, LiteLLM 4000, Open WebUI 3000,
  landing 9091/9092/9102, BTCPay 8500, membership-sync 9101, Minecraft
  25565) is now documented as a stable convention across installs.
- Data subpaths under `${LS_DATA_DIR}` are preserved from the original
  deployment, so an existing instance migrates by setting the `LS_*`
  variables to its current locations — volumes are found in place, no data
  moves.

### Security

- TLS terminates only on the operator's node; the VPS relays ciphertext by
  SNI and holds no keys, certificates, or plaintext.
- Synapse admin API blocked at the Caddy edge; all services loopback-bound
  except explicit WireGuard stream binds; security headers applied at the
  edge; no inline secrets anywhere in the distribution.
